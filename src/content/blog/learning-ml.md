I started with C++ and low-level systems programming — understanding memory, pointers, how a computer actually executes instructions. That foundation made the jump to machine learning less intimidating than it looks from the outside.

Python is the lingua franca of ML, and it was my third language after C++ and Java. The syntax felt loose at first, almost too simple. But the ecosystem is where the power lives: NumPy for linear algebra, PyTorch for building models, scikit-learn for classical algorithms. You're not writing boilerplate — you're expressing mathematical operations directly.

My first real project was a face detection system built on top of OpenCV and a custom CNN. It ran on a distributed architecture with Docker containers, which meant I got to combine my ML knowledge with DevOps skills. That intersection — AI systems that actually deploy and scale in production — is where I want to focus my career.

The hardest part isn't the code. It's understanding *why* a model makes the decisions it does. Gradient descent, backpropagation, loss functions — these are mathematical concepts that require patience to internalize. But once they click, you see the world differently.
